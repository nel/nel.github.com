Un nouveau défi en Ruby !

Ca y est, le DIMELO Contest est de retour.

Pour cette 3e édition, le principe est simple : VOUS faites (toujours) avancer la communauté RUBY et DIMELO vous récompense.

Comment faire ? Les détails et modalités de participation, sont ici.

Peu de règles (comme d'habitude)

Tout rubyiste peut participer. Pour gagner, il faut :

  • écrire un patch pour un projet open-source en ruby hébergé sur github et possédant au moins 20 "Stars"
  • soumettre une pull request auprès des mainteneurs du projet
  • obtenir la validation de sa pull request avant le 3 janvier 2013 à minuit CEST
  • nous envoyer la référence de la pull request à contest at dimelo.com, toujours avant le 3 janvier 2013 à minuit CEST.

Les 3 gagnants seront tirés au sort lors du premier événement Paris.rb de 2013

Des questions ? Envoyez-nous un tweet @dimelo sans oublier le hashtag #dimelocontest.

Sinon, rendez-vous en section contact.

Les détails du défi

Les projets open-source acceptés pour ce défi

Les projets open-source hébergés sur github

Pour permettre à plus de Rubyistes de participer nous avons décidé d'étendre la liste des projets qui sont acceptés. Nous accepterons les pull-requests effectuées sur des projets Ruby avec plus de 20 "Stars" sur GitHub. Cela fonctionne même si c'est votre propre projet, le compte github qui fait la pull-request doit exister AVANT le début de ce contest.

Le tirage au sort

Comment allons-nous déterminer les gagnants ?

Seules seront prises en compte les pull request acceptées dans la liste des projets. Si vous pensez qu'un autre projet Ruby mérite d'être dans la liste, faites nous en part et nous le rajouterons à la liste des projets propososés par la communauté.

Queqlues précisions sur le tirage au sort

  • Les gagnants seront tirés au sort parmi les personnes qui auront vu leur pull request acceptée pendant la durée du concours.
  • Dans un souci d'équité, chaque participant au défi n'a qu'une seule chance de gagner l'une des trois tablettes Nexus, et ce même si plusieurs de ces pull requests ont été acceptées pendant la durée du DIMELO Contest. Pour mettre toutes les chances de votre côté, nous vous encourageons tout de même à soumettre plusieurs pull requests.
  • La pull request doit être du code Ruby (la documentation ne sera pas prise en compte).
  • Les pull requests rejetées ne seront pas prises en compte.
  • Seules les pull requests effectuées à partir du 10 décembre 2012 à midi CEST seront prises en compte.
  • Coup de pouce : dans l'hypothèse où votre pull request ne serait pas acceptée par les mainteneurs dans le délai imparti, vous pouvez quand même nous la faire parvenir et notre jury l'analysera et validera le cas échéant votre participation au tirage au sort. Bien entendu, les membres du jury et leurs collègues ne peuvent pas participer au concours.
